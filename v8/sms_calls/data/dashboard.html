Block lists

One number (or sender id like JAZZ) per line. Incoming = still logged but no ntfy notification. Outgoing = the device refuses to send/dial; the job is marked blocked. Changes auto-apply on the device within ~1 minute, or instantly after pressing Sync.

WiFi Networks

Saved to Firebase; the device reads them on sync and stores them locally. On reboot it tries pair 1, then pair 2, then the networks in secrets.h, then its own AP. Set these while the device is online, then reboot.

App developer guide

Queue an SMS (one document per number)

setDoc(doc(db, 'sim_module/sms/sms_jobs', '+923001234567'), {
  message: 'hello', status: 'pending', enque_by: 'app:orders'
})

Queue a missed call

setDoc(doc(db, 'sim_module/calls/call_jobs', '+923001234567'), {
  status: 'pending', enque_by: 'app:otp'
})

The device updates status

pending → in_progress → sent / called / blocked / failed. blocked means the number is in the outgoing block list. enque_by is preserved so you can link replies back to the sender.

Read incoming

sim_module/sms/sms_received/{number}    // normalized_message, notified, blocked
sim_module/calls/call_received/{number} // notified, blocked
